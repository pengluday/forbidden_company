# PRD: 小红书浏览器插件采集器（免费版 MVP）

## 1. 背景与问题
小红书笔记评论是重要线索来源，但直接做中心化爬虫会带来明显的风控问题，尤其是 `IP` 封禁、账号异常、接口失效等风险。

当前更合适的产品形态不是“平台代抓”，而是“用户在自己的浏览器里主动采集”。这样可以：
- 将采集动作限制在用户主动点击的前提下
- 避免把高频抓取集中到单一服务器 `IP`
- 复用用户自己的登录态，降低匿名接口失败率
- 让产品更容易做成免费版，再逐步扩展到付费版

## 2. 产品目标
为用户提供一个浏览器插件，让其在打开小红书笔记页面时，可以一键采集：
- 帖子正文
- 评论内容
- 评论作者、时间、点赞数等基础元数据

采集结果应能够：
- 在本地查看
- 导出为 CSV / JSON
- 后续接入现有目录系统或本地后端

## 3. 产品定位
- 产品类型：浏览器插件 + 本机采集辅助服务
- 核心价值：
  - 把“采集入口”前置到浏览器
  - 把“抓取和分页逻辑”保留在本机侧
  - 用更低的使用门槛换取更高的可用性
- 非目标：
  - 不做平台对抗式绕过
  - 不做云端代抓
  - 不做自动化批量扫站
  - 不做违法、骚扰、恶意用途支持

## 4. 目标用户
- 关注职场公平、年龄歧视议题的普通用户
- 希望整理小红书线索的研究者或内容整理者
- 后续愿意为更高效率和更强管理能力付费的重度用户

## 5. 用户故事
1. 作为用户，我在小红书笔记页面上点击插件按钮，就能采集当前帖子和评论。
2. 作为用户，我希望看到采集进度和成功/失败原因。
3. 作为用户，我希望把结果导出成 CSV 或 JSON 继续整理。
4. 作为用户，我希望免费版就能完成基本采集，不需要登录复杂后台。
5. 作为后续付费用户，我希望管理多个项目、保存历史、批量处理链接。

## 6. MVP 范围
### 6.1 免费版必须做
- 浏览器插件入口
- 当前页面 URL 识别
- 一键采集当前笔记的帖子正文
- 一键采集当前笔记的评论
- 显示采集结果数量
- 本地导出 CSV / JSON
- 本地保存基础配置

### 6.2 免费版暂不做
- 云端账户系统
- 付费订阅
- 团队协作
- 批量历史管理
- 自动定时采集
- 站外全站搜索

### 6.3 付费版后续可做
- 多链接批量队列
- 项目分组
- 历史记录同步
- 高级筛选与去重
- 报表和标签系统
- 更细的配额控制

## 7. 关键用户流程
1. 用户打开小红书笔记页面。
2. 用户点击浏览器插件。
3. 插件识别当前 URL，并把采集任务发给本机辅助服务。
4. 本机服务使用浏览器会话、cookie 和分页逻辑采集帖子与评论。
5. 插件展示结果摘要。
6. 用户导出结果或继续采集下一条。

## 8. 功能需求
### FR-1 当前页面识别
- 插件必须识别当前小红书笔记链接
- 插件必须支持用户手动修改 URL

### FR-2 帖子采集
- 采集帖子标题、正文、发布时间、来源链接
- 若正文为空，至少应保留可追溯标题和来源链接

### FR-3 评论采集
- 采集评论正文
- 采集评论作者、点赞数、创建时间等基础字段
- 支持分页翻页，不能只停留在第一页
- `comment_limit=0` 表示抓取全部可见评论

### FR-4 结果展示
- 展示帖子条数
- 展示评论条数
- 展示是否采集成功
- 展示失败原因

### FR-5 导出
- 支持导出 CSV
- 支持导出 JSON
- 支持复制结果摘要

### FR-6 本地配置
- 支持保存本机后端地址
- 支持保存 cookie 或由本机服务自动读取
- 支持配置默认评论上限

## 9. 非功能需求
- NFR-1: 插件交互应在 2 秒内给出开始反馈
- NFR-2: 不应阻塞浏览器主线程
- NFR-3: 采集过程应尽量透明，用户可见状态更新
- NFR-4: 允许不同用户使用各自浏览器会话，减少集中 `IP` 风险
- NFR-5: 设计上应支持后续接入付费授权和配额

## 10. 架构建议
### 推荐方案
- `Chrome Extension`
  - 负责入口、UI、参数配置、结果展示
- `本机 Companion Service`
  - 负责真实抓取、分页、cookie 读取、导出、去重
- `可选云端服务`
  - 只负责账号、授权、配额、支付，不直接代抓

### 为什么这样设计
- 插件本身更轻
- 抓取逻辑可以复用现有本机实现
- 风控压力分散到用户自己的浏览器环境
- 后续免费版和付费版都能共用同一套抓取核心

## 11. 风控与合规
- 不承诺规避平台风控
- 不做平台对抗式自动化
- 不把“匿名大规模抓取”作为卖点
- 明确提示用户自行遵守平台规则和当地法律
- 对采集结果应保留来源链接和采集时间，避免绝对化表述

## 12. 免费版策略
免费版建议只提供：
- 单条页面采集
- 基础导出
- 本机本地保存
- 基础设置项

免费版的目标不是“做大流量”，而是验证：
- 用户是否愿意在自己的浏览器里一键采集
- 评论页是否足够有价值
- 用户是否愿意为批量和管理能力付费

## 13. 付费版方向
如果免费版验证成立，付费版可以围绕以下能力收费：
- 批量队列
- 历史同步
- 高级筛选
- 项目管理
- 团队协作
- 使用配额

## 14. 成功指标
- 用户从打开页面到完成首次采集的时间短于 1 分钟
- 单条页面采集成功率可稳定达到可用水平
- 评论采集能覆盖第一页之外的分页内容
- 免费版能支撑真实用户进行连续试用
- 有足够用户愿意进一步使用批量版或管理版

## 15. 主要风险
- 小红书页面结构变化导致解析失效
- 平台风控升级导致 cookie 失效或评论失败
- 用户本机环境差异导致插件兼容问题
- 插件被误解为“自动化爬虫”而引发合规风险
- 免费版和付费版边界不清，导致产品预期混乱

## 16. 验收标准
- 插件可在小红书笔记页面上启动
- 可采集帖子正文
- 可采集评论且支持分页
- 可导出结果
- 可在本机服务中保存采集记录
- 免费版功能闭环完整，可供真实用户试用

## 17. 下一步建议
1. 先做插件 MVP 的信息架构和交互草图
2. 再定义本机 Companion Service 的 API
3. 最后拆分免费版与付费版能力边界

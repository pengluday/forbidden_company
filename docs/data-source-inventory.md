# BMAD Data Source Inventory (MVP -> V1)

## 目标
把“35+年龄歧视公司目录”的数据来源拆成可逐项落地的采集清单，便于你按优先级逐个执行。

## Source 01: 招聘网站职位描述（最高优先级）
- 典型平台：BOSS直聘、智联招聘、前程无忧、猎聘、拉勾
- 价值：最直接证据，容易出现“35岁以下/年龄限制”等描述
- 采集方式：人工检索 + 截图留存 + 原链接归档
- 检索关键词：`35岁以下` `年龄限制` `年龄要求` `年轻化团队`
- 必采字段：公司名、岗位名、城市、发布时间、原文片段、URL、截图路径
- 核验规则：同公司出现2条以上相关岗位，判定为`partial`
- 风险提示：职位可能下架，必须保留截图和采集时间



## Source 02: 小红书用户线索
- 价值：发现线索快，覆盖面广
- 采集方式：人工检索帖子和评论，记录帖文URL、评论片段与发布时间
- 检索关键词：`35+ 面试` `年龄卡` `公司名 + 招聘`
- 必采字段：帖子URL、作者ID(脱敏)、发布时间、截图、涉及公司、原话摘要、评论摘录
- 核验规则：仅可作为“线索来源”，必须配合其他来源交叉验证
- 风险提示：高噪音，禁止单一帖子直接定性公司

## Source 03: 抖音用户线索
- 价值：高传播，能快速发现热点公司
- 采集方式：关键词检索 + 视频链接 + 评论区信息摘录
- 检索关键词：`35岁 找工作` `年龄歧视` `公司名`
- 必采字段：视频URL、发布时间、作者ID(脱敏)、关键内容摘要、截图
- 核验规则：与招聘信息/新闻互证后才升级证据等级
- 风险提示：转述内容多，需谨慎防误伤

## Source 04: 脉脉/职场社区爆料
- 价值：常含内部招聘流程信息
- 采集方式：按公司名和岗位关键词检索
- 检索关键词：`公司名 + 35` `年龄` `面试`
- 必采字段：帖子链接、时间、涉及业务线、证据摘要、截图
- 核验规则：默认线索级，不单独作为结论证据
- 风险提示：匿名内容可信度波动大

## Source 05: 黑猫投诉/消费投诉平台（辅助）
- 价值：可发现雇主品牌与劳动争议交叉问题
- 采集方式：公司名检索 + 主题筛选
- 检索关键词：`公司名 + 招聘` `公司名 + 歧视`
- 必采字段：投诉链接、投诉时间、问题类型、摘要
- 核验规则：仅辅助画像，不作为核心证据
- 风险提示：主观描述多，不能单独判断





## 落地优先级（建议）
1. Source 01 招聘网站职位描述
4. Source 02 小红书
5. Source 03 抖音
8. Source 04 脉脉/社区
10. Source 05 投诉平台

## 入库最小标准（MVP）
- 至少1条可追溯链接 + 1份截图
- 明确公司主体（非模糊简称）
- 有采集日期与证据摘要
- 标记来源类型与核验状态（pending/partial/verified）

## 证据分级建议
- L1 线索：单条社区内容（小红书/抖音/社区）
- L2 交叉：社区 + 招聘网站 或 社区 + 新闻
- L3 高可信：新闻/监管/裁判文书 + 招聘证据可互证

# 流程地图

## 1. 整体链路
1. 用户提报线索或产品建议
2. 进入 SQLite 待审核队列
3. 后台人工核验
4. 通过后进入正式表
5. 前台自动读取最新数据

## 2. 主要卡点
1. 服务启动顺序
2. 需要人工审核，不可能完全自动
3. 产品映射和线索核验都可能积压
4. 前台如果只读静态 JSON，会和后台脱节

## 3. 一键入口
1. `scripts/dev_up.sh`
2. 作用：初始化数据库、导出前台数据、启动后台和前台
3. 访问：前台 `8080`，后台 `8787`
4. 后台里可以直接点“抓取智联最新100条列表”

## 4. 批量入口
1. `scripts/batch_verify_collect.py`
2. `scripts/batch_approve_products.py`
3. `scripts/run_zhilian_collection.sh`
4. `scripts/run_xiaohongshu_collection.sh`
5. 适合一次性处理一批采集或审核结果，智联和小红书都可以批量跑

## 5. 建议工作方式
1. 日常用后台页面做小批量审核
2. 大量积压时用批处理脚本
3. 前台用户提交会自动进入待审核状态
4. 后台审核通过后自动同步到前台

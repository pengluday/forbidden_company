# 小红书浏览器插件 MVP 页面结构与交互草稿

## 1. 设计目标
插件的目标不是“堆功能”，而是让用户在 1 分钟内完成一次完整采集：
1. 打开小红书笔记页面
2. 点击插件
3. 一键抓取帖子和评论
4. 看见结果
5. 导出或继续下一条

页面设计要满足：
- 低学习成本
- 低打扰
- 状态清晰
- 失败可恢复
- 免费版也能闭环

## 2. 页面形态
建议做成两层：

### 2.1 浏览器侧入口
- Toolbar icon
- 点击后打开 popup
- 可选：右侧 side panel

### 2.2 插件主面板
建议优先用 `popup` 做 MVP。
如果后续需要更长的结果列表和任务历史，再升级成 `side panel`。

## 3. 信息架构
主面板从上到下分 5 块：
1. 连接状态
2. 当前页面信息
3. 采集控制
4. 结果摘要
5. 最近任务 / 导出入口

## 4. 主面板结构
### 4.1 顶部状态条
内容：
- 插件名称
- 本机服务连接状态
- 当前是否在小红书页面

状态样式：
- `在线`：绿色
- `离线`：灰色或红色
- `采集中`：蓝色加载态

### 4.2 当前页面卡片
展示：
- 页面标题
- 当前 URL
- 是否识别为小红书笔记

交互：
- `自动识别` 当前标签页 URL
- `手动修改` URL 输入框
- `粘贴` 当前链接按钮

### 4.3 采集控制区
建议只保留最必要的选项：
- `抓取帖子 + 评论` 主按钮
- `评论上限`
- `0=全部` 的说明
- `只抓帖子` 作为次要操作

免费版默认：
- `评论上限 = 0`
- `include_comments = true`

### 4.4 结果摘要区
展示：
- 帖子数
- 评论数
- 是否成功
- 耗时
- 最近错误信息

推荐布局：
- 大号数字显示评论条数
- 小字显示 URL 和时间
- 错误状态用可读文案，而不是原始栈信息

### 4.5 导出与历史区
展示：
- `导出 CSV`
- `导出 JSON`
- `复制摘要`
- `最近 5 条任务`

历史项建议显示：
- 标题截断
- 任务状态
- 条数
- 时间

## 6. 主要交互流程
### 6.1 首次使用
1. 用户安装插件
2. 打开任意小红书笔记页
3. 点击插件
4. 插件检测本机服务是否在线
5. 若未配置，提示先填写本机服务地址
6. 若在线，显示当前 URL 和采集按钮

### 6.2 单条采集
1. 用户点击 `抓取帖子 + 评论`
2. 按钮进入 loading
3. 结果区实时显示 `采集中`
4. 完成后显示条数与摘要
5. 用户可立即导出

### 6.3 失败重试
1. 若评论接口异常，显示 `登录态失效` 或 `页面异常`
2. 提示用户重新登录或刷新 cookie
3. 用户点击重试

### 6.4 导出
1. 用户点击 CSV / JSON
2. 插件请求本机服务生成文件
3. 浏览器开始下载

## 7. 状态设计
### 7.1 空态
场景：
- 未打开小红书页面
- 本机服务未配置

提示文案：
- “打开一条小红书笔记后即可开始采集”
- “请先连接本机服务”

### 7.2 采集中
显示：
- loading spinner
- 当前阶段：准备中 / 抓帖子 / 抓评论 / 写入结果

### 7.3 成功态
显示：
- 帖子条数
- 评论条数
- 导出按钮高亮

### 7.4 失败态
显示：
- 失败原因
- 重试按钮
- 查看日志按钮

## 8. 文案建议
### 推荐文案
- 主按钮：`抓取帖子 + 评论`
- 次按钮：`只抓帖子`
- 说明：`评论上限，0=全部`
- 成功：`采集完成，可导出`
- 失败：`采集失败，点击重试`

### 避免文案
- 不要写“无敌抓取”
- 不要写“绕过风控”
- 不要写“自动爬全站”
- 不要写“永久可用”

## 9. 视觉方向
MVP 不需要复杂视觉，但建议做到：
- 信息层级清晰
- 主按钮明显
- 状态颜色分明
- 错误提示不刺眼
- 支持暗色 / 亮色两种浏览器主题适配

建议风格：
- 背景：浅色中性
- 主色：深蓝或墨绿，避免过度互联网化
- 字体：系统字体即可，优先清晰而不是花哨
- 图标：简洁线性图标

- `TaskHistory`

### 可选组件
- `LoginCookieHint`
- `RetryPanel`
- `UsageTip`

## 11. 可扩展布局
后续如果做付费版，可以加：
- 项目选择器
- 批量队列
- 标签
- 历史过滤
- 使用额度条

建议这些都放在主面板的底部或二级页，不要抢占单条采集的主路径。

## 12. MVP 验收
这版 UI 合格的标准是：
- 用户能一眼看出插件是否在线
- 用户能在 3 次点击内开始采集
- 结果区能明确知道抓到了多少帖子和评论
- 用户能导出结果
- 用户能知道为什么失败
